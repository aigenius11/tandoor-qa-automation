 # Дипломный проект: Автоматизация тестирования Tandoor Recipes

## 1. Описание проекта
Данный проект представляет собой комплекс автотестов для веб-приложения по управлению рецептами Tandoor Recipes. Работа выполнена в рамках дипломного проекта по курсу QA Automation. Основная цель — автоматизация критических сценариев пользовательского пути (UI и API) и настройка процесса непрерывной интеграции (CI).

Технологический стек:
* Язык: Python 3.10
* Фреймворк: Pytest
* Автоматизация UI: Selenium WebDriver
* Паттерн: Page Object Model (POM)
* CI/CD: GitHub Actions
* Отчетность: Allure Framework

## 2. Требования для запуска
Для корректной работы проекта необходимо наличие:
* Python версии 3.10.
* Менеджера пакетов pip.
* Установленного Google Chrome (используется WebDriver Manager для автоматической подгрузки драйвера).

## 3. Инструкции по установке и запуску
1. Клонируйте репозиторий:
git clone https://github.com/aigenius11/tandoor-qa-automation.git
2. Перейдите в папку проекта:
cd qa_auto_tandoor
3. Установите зависимости:
pip install -r requirements.txt
4. Запустите тесты:
pytest --alluredir=allure-results

## 4. Структура проекта
* .github/workflows/ — конфигурация CI/CD для автоматического запуска тестов в GitHub Actions.
* api/ — модули для взаимодействия с API приложения (включая client.py).
* pages/ — реализация паттерна Page Object. Содержит логику работы со страницами (Login, Recipes, Meal Plan, Shopping List).
* tests/ — наборы тестовых сценариев (UI тесты, API тесты, E2E сценарии).
* screenshots/ — скриншоты результатов тестирования и ошибок.
* conftest.py — конфигурационный файл pytest с фикстурами.
* requirements.txt — список всех внешних зависимостей.

## 5. Реализованные сценарии тестирования
Автоматизированы следующие ключевые области:
* Авторизация (UI): валидный вход и проверка перехода в Dashboard.
* API-клиент: проверка взаимодействия с бэкендом через токены.
* Управление рецептами: создание тестовых данных и проверка их отображения.
* Shopping List & Meal Plan: комплексные тесты на добавление продуктов и планирование питания.
* E2E сценарии: сквозные проверки функциональности приложения от логина до результата.

## 6. Список используемых библиотек
* pytest — базовый фреймворк для тестирования.
* selenium — управление браузером и элементами UI.
* webdriver-manager — автоматическая установка ChromeDriver.
* allure-pytest — интеграция с системой отчетности Allure.
* python-dotenv — управление секретными данными через .env.
* requests — для реализации API тестов.

## 7. Инструкция по получению API-токена
1. Авторизуйтесь в приложении Tandoor Recipes.
2. Перейдите в User Settings (Настройки пользователя).
3. Откройте раздел Security или API Keys.
4. Создайте новый ключ (Token), скопируйте его для использования в тестах.

## 8. Пример файла с переменными окружения (.env)
Для локального запуска создайте файл .env в корне проекта:
TANDOOR_USERNAME=ваш_логин
TANDOOR_PASSWORD=ваш_пароль
TANDOOR_API_TOKEN=ваш_токен_из_пункта_7
